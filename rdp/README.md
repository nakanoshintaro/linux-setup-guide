# RDP（Ubuntu標準のリモートデスクトップ）

## 概要

Ubuntu 24.04 には、標準でリモートデスクトップ機能が搭載されている。  
この機能を使うことで、Windows から Ubuntu に対して GUI で接続できる。

本手順では、**Ubuntu標準搭載のリモートデスクトップ機能**を使い、  
**Windows の「リモート デスクトップ接続」アプリから Ubuntu に接続する方法**をまとめる。

---

## 特徴

### 長所

- Ubuntuに標準搭載されているため、追加のRDPサーバが不要
- Windows標準のRDPクライアントから接続可能
- GUI設定中心で導入しやすい

### 注意点

- `xrdp` とは別物
- 接続時は「画面共有」ではなく、セッション切替に近い挙動になる
- ローカル側のユーザーが切断されることがある

---

## xrdpとの違い

| 項目 | Ubuntu標準RDP | xrdp |
|------|--------------|------|
| 導入 | 不要（標準） | 必要 |
| 設定 | GUI中心 | 設定ファイル |
| 挙動 | セッション切替 | 別セッション |

---

## 構成

```text
Windows ── RDP ── Ubuntu
```

---

## 前提条件

- Ubuntu 24.04
- Windows PC
- 同一ネットワーク内

---

## 手順

### 1. リモートデスクトップを有効化

1. 「設定」を開く  
2. 「システム」→「リモートデスクトップ」  
3. 「Desktop Sharing」をON  
4. 「リモートコントロール」をON  

---

### 2. IPアドレス確認

```bash
hostname -I
```

例:

```text
192.168.1.7
```

---

### 3. ポート開放

```bash
sudo ufw allow 3389/tcp
```

確認:

```bash
sudo ufw status
```

---

### 4. Windowsから接続

1. 「リモート デスクトップ接続」を起動（mstsc）
2. IPアドレス入力

```text
192.168.1.7
```

3. 接続
4. ユーザー名・パスワード入力

---

## 接続時の挙動

- ローカルユーザーが切断される場合あり
- 同時に同じ画面は見れない
- 画面共有ではなく「ログイン制御」に近い

---

## トラブルシューティング

### 接続できない

#### ping確認

```cmd
ping 192.168.1.7
```

---

#### ポート確認

```bash
ss -tuln | grep 3389
```

---

#### 設定確認

- Desktop Sharing → ON
- リモートコントロール → ON

---

### 3390が表示される場合

```bash
ss -tuln | grep 339
```

必要なら開放:

```bash
sudo ufw allow 3390/tcp
```

---

### セッションが切れる

仕様。  
画面共有したい場合はVNCを使用する。

---

## よくある疑問

### Remminaは必要？

不要（Linux側クライアント用）

---

### xrdpは必要？

不要（標準機能で対応可能）

---

### なぜ画面共有にならない？

RDPはログイン制御系のプロトコルのため

- 3389/tcpを開放する
- 完全な画面共有ではない
- 画面共有目的ならVNCを使う
